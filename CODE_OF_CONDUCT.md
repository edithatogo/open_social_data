# Code of Conduct

## Our Pledge

We are committed to a respectful, inclusive, and practical project environment. The goal of Open Social Datasets is to make public social data easier to use, and that requires collaboration across different backgrounds and skill levels.

## Expected Behaviour

Participants are expected to:

* Be respectful and constructive.
* Assume good faith, while still being clear about technical or factual problems.
* Welcome questions from people with different levels of data, coding, policy, or domain experience.
* Credit source agencies and contributors accurately.
* Discuss data limitations, uncertainty, and caveats honestly.
* Keep disagreements focused on evidence, reproducibility, and project goals.

## Unacceptable Behaviour

Unacceptable behaviour includes:

* Harassment, intimidation, threats, or personal attacks.
* Discriminatory language or conduct.
* Sexualised language or unwanted attention.
* Publishing private information without consent.
* Sharing credentials, tokens, private data, or restricted datasets.
* Misrepresenting source data, removing caveats in a misleading way, or knowingly publishing false documentation.
* Repeated disruption after a maintainer asks for a behaviour to stop.

## Data Responsibility

This project works with social data. Contributors must treat privacy, suppression, Indigenous data considerations, source licences, and official caveats seriously. If a source marks data as provisional, suppressed, rounded, or otherwise qualified, repository documentation and examples should preserve that context.

## Enforcement

Maintainers may take action that is appropriate to the situation, including:

* Asking for clarification or correction.
* Editing or removing inappropriate content.
* Closing issues or pull requests.
* Temporarily or permanently restricting participation.

Enforcement should be proportionate, documented where practical, and focused on protecting the project and its contributors.

## Reporting

Report conduct concerns through the repository issue tracker if public reporting is appropriate. If the concern involves private information, safety, harassment, or sensitive data, contact a maintainer privately instead of posting details publicly.

Include enough information for maintainers to understand what happened, where it happened, and what outcome is needed. Do not include sensitive personal information unless it is necessary and safe to share with the maintainer.

## Scope

This code of conduct applies to project repositories, issues, pull requests, documentation, discussions, and any project-related communication space.

## Attribution

This code is adapted for the Open Social Datasets project and draws on common open source community conduct principles.
