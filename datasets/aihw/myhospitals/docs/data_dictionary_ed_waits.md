# Data Dictionary for MyHospitals - ED Waiting Times (MYH-ED-WAITS)

**Dataset Source:** Australian Institute of Health and Welfare (AIHW) - MyHospitals API (`/flat-formatted-data-extract/MYH-ED-WAITS`)
**Date of Dictionary Creation/Update:** $(date +"%Y-%m-%d")

---

## 1. Introduction

This data dictionary describes the variables within the "Emergency Department (ED) Waiting Times" dataset, specifically from the `MYH-ED-WAITS` measure category, accessed via the AIHW MyHospitals API's `/flat-formatted-data-extract/` endpoint. The data is processed and stored in this repository (planned format: Parquet).

The purpose of this document is to help users understand the structure, content, and meaning of each variable. The fields are based on the `FormattedDataExtractModel` schema from the API's Swagger definition and observed sample data.

## 3. Code Lists / Classifications

*This section will be expanded as more measure categories and their specific codelists are explored. For now, it highlights general types of codes found.*

### 3.1. `measure_code` / `reported_measure_code`
*   These are specific codes defined by AIHW for each measure and its disaggregations. A full list can be obtained by querying relevant metadata endpoints (e.g., `/measures`, `/reported-measures`).

### 3.2. `reporting_unit_code`
*   Codes for hospitals, Local Hospital Networks (LHNs), Primary Health Networks (PHNs), States etc.
*   Example: `H0014` (The Children's Hospital at Westmead)

### 3.3. `reporting_unit_type_code`
*   Example: `H` (Hospital), `LHN` (Local Hospital Network), `PHN` (Primary Health Network), `STATE` (State/Territory).

### 3.4. `peer_group_code`
*   Codes representing hospital peer groups.
*   Example: `CH` (Children's hospitals), `MJ` (Major hospitals), `MM` (Medium metropolitan hospitals), `LM` (Large metropolitan hospitals), `U` (Unpeered).

### 3.5. `reported_measure_category_code` (for ED Waiting Times)
*   Codes representing triage categories for ED Waiting Times.
*   Example: `ED_Resuscitation`, `ED_Emergency`, `ED_Urgent`, `ED_SemiUrgent`, `ED_NonUrgent`.

### 3.6. `caveat_codes`
*   Codes for specific data caveats (e.g., data quality notes, suppression reasons). Details can be fetched from the `/caveats/{caveat-code}` endpoint.
