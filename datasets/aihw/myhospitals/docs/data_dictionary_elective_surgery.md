**Date of Dictionary Creation/Update:** $(date +"%Y-%m-%d")

---

## 1. Introduction

This data dictionary describes the variables within the "Elective Surgery" dataset, specifically from the `MYH-ES` measure category, accessed via the AIHW MyHospitals API's `/flat-formatted-data-extract/` endpoint. The data is processed and stored in this repository (planned format: Parquet).

The purpose of this document is to help users understand the structure, content, and meaning of each variable. The fields are based on the `FormattedDataExtractModel` schema from the API's Swagger definition and observed sample data. The column structure is consistent with other flat formatted data extracts from this API.

## 3. Code Lists / Classifications (Examples for MYH-ES)

*This section will be expanded as more specific codelists are analyzed from the data or API metadata endpoints.*

### 3.1. `measure_code` / `reported_measure_code` (within MYH-ES)
*   Example `measure_code`: `MYH0006` ("Number of elective surgeries"), `MYH0007` ("Waiting time for elective surgery - 50th percentile"), `MYH0008` ("Waiting time for elective surgery - 90th percentile").
*   Example `reported_measure_code`: `MYH-RM0044` ("Urgent elective surgery" - disaggregation of MYH0006).

### 3.2. `reporting_unit_code`
*   Codes for hospitals. Example: `H0014` (The Children's Hospital at Westmead).

### 3.3. `reported_measure_category_code` (Disaggregation for MYH-ES)
*   Primarily by "Urgency categorisation". Example: `ES_U1_Urgent`, `ES_U2_SemiUrgent`, `ES_U3_NonUrgent`.
*   May also include disaggregations by surgical specialty for some measures.

### 3.4. `reported_measure_category_type_code`
*   Example: `ES_URGENCY` (Indicates the disaggregation is by urgency category).

*Refer to AIHW MyHospitals API metadata endpoints (e.g., `/measures`, `/reported-measures`, `/reporting-units`) for comprehensive lists of codes.*
