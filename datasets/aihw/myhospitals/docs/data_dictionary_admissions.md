**Dataset Source:** Australian Institute of Health and Welfare (AIHW) - MyHospitals API (`/flat-formatted-data-extract/MYH-ADM`)
**Date of Dictionary Creation/Update:** $(date +"%Y-%m-%d")

---

## 1. Introduction

This data dictionary describes the variables within the "Admissions" dataset, specifically from the `MYH-ADM` measure category, accessed via the AIHW MyHospitals API's `/flat-formatted-data-extract/` endpoint. The data is processed and stored in this repository (planned format: Parquet).

The purpose of this document is to help users understand the structure, content, and meaning of each variable. The fields are based on the `FormattedDataExtractModel` schema from the API's Swagger definition and observed sample data. The column structure is consistent with other flat formatted data extracts from this API (e.g., ED Waiting Times).

## 3. Code Lists / Classifications (Examples for MYH-ADM)

*This section will be expanded as more specific codelists are analyzed from the data or API metadata endpoints.*

### 3.1. `measure_code` / `reported_measure_code` (within MYH-ADM)
*   Example `measure_code`: `MYH0024` ("Number of admissions to hospital")
*   Example `reported_measure_code`: `MYH-RM0213` ("Medical (emergency)" as a type of care)
*   Other reported measures will exist for different types of care (e.g., surgical, maternity, mental health).

### 3.2. `reporting_unit_code`
*   Codes for hospitals. Example: `H0012` (State Forensic Mental Health Service), `H0014` (The Children's Hospital at Westmead).

### 3.3. `reported_measure_category_code` (Disaggregation for MYH-ADM)
*   Primarily by "Type of care". Example: `APC_MedicalEmergency`, `APC_Surgical`, `APC_Obstetrics`, `APC_MentalHealth`.

### 3.4. `reported_measure_category_type_code`
*   Example: `CARE_TYPE` (Indicates the disaggregation is by type of care).

*Refer to AIHW MyHospitals API metadata endpoints (e.g., `/measures`, `/reported-measures`, `/reporting-units`) for comprehensive lists of codes.*
